    Introduction

    Welcome to the L.cash API! You can use the L.cash payment system in various languages using the API.

    You can use the L.cash payment platform in various languages such as Sell, Node, PHP and JavaScript. You can see demo code examples in the area on the right and you can switch programming languages with the tabs at the top.

    1. The interface adopts UTF-8 character set encoding.

    2. The interface adopts https protocol

    3. The interface access frequency is limited to background configuration.

    You need to replace the APP KEY and APP SECRET here.

    livilpay uses API keys to allow access to the API. You can register a new livilpay API key at our developer portal.

    Request signature, all parameter pairs are sorted in ascending order by dictionary (abcd ...), and the secret assigned by the platform is the key. First, all parameters except the signature itself need to be added in the format "KEY = value" and use the character "&". Add the exact order of the request, then use md5 to encrypt the generated string and then lowercase it. Note: The sign parameter does not participate in the signature; the null parameter does not participate in the signature.

    Request header

    Parameter Default Description
    Content-Type String application/json;charset=utf8
    Accept-Language String Client language type zh-CN: Chinese, en-US: English, ko-KR: Korean

    Query Parameters

    Parameter Default Description
    amount Float Fiat currency payment amount
    merchantOrderNo String(*) Merchant order number（unique）
    goodsName String product name
    ts Long(*) Timestamp (seconds)
    appKey String(*) Interface unique private appKey.
    sign String(*) Request a signature, and all parameter pairs are sorted in ascending dictionary order (for example: a=c&b=g&F=j...), add &APPSECRET=key at the end of the sorting. When sorting, you need to convert the parameter name to uppercase, and then use md5 for the string Encrypt, then convert it to lowercase

    Response format

    Parameter Default Description
    code Integer Return code 200: success 201: Failure
    msg String Description
    data Object/Array Return data, object or array { "Code": 0, "Msg": "success", data: {} }

    Errors

    The API uses the following error codes:

    Code Meaning
    200 successful.
    201 The operation failed.
    202 data is abnormal.
    203 Signature verification failed.
    206 The number of visits exceeds the limit, and the visit is denied.
    207 No permission to access.
    208 The request expired.
    400 The request parameter is wrong.
    405 The account is disabled.
    407 The client type verification failed
    408 The payment password is incorrect.
    415 Unsupported Media Type.
    500 Service call exception.
