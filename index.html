Sign(API Access Signature)

Request data signature instructions

Request parameters are passed in JSON format, parameter example:

{"appKey": "MK167556","cardHolderId": 1,"cardNo": "5554748800101621","sign": "98e243ec0a837e693ea0d832ee9facb1","ts": 1675444481634}

appKey: APPKey applied by the enterprise

Keysign: request signature

ts: current timestamp (UTC time)

Test host: http://47.241.221.172:8107

Test appKey: MK167556

Test appSecret: 7ymscjnm6hmpqikxtzl1xi7zi

Perform natural ascending sort splicing according to parameters and convert parameter names to uppercase

APPKEY=MK167556&CARDHOLDERID=1&CARDNO5554748800101621&TS=1675444481634

Then concatenate the key at the end of the string(&APPSECRET=7ymscinm6hmpqikxtzl1xj7zj)

APPKEY-MK167556&CARDHOLDERID-1&CARDNO-5554748800101621&1S=1675444481634KAPPSECRET=7ymsc inm6hmpgikxtz11xi7zi

Then encrypt the string with MD5 (32 bits)

WD5(APPKE-K16556KCARDHLDER1-1KARDNJ-5474800101621KS-1654481634PPSECRE- msc m6mpikxt21lx zi)-> 98e24eca837e693eadiS32ee9facb

Get the final call interface parameters

"appkey":"K167556","cardHolderld": l,"cardNo": "554748800101621" "sign": "98e243eca837e693ead832ee9facbl" "ts": 1675444481634


*Request a signature, and all parameter pairs are sorted in ascending dictionary order (for example: a=c&b=g&F=j...), add &APPSECRET=key at the end of the sorting. When sorting, you need to convert the parameter name to uppercase, and then use md5 for the string Encrypt, then convert it to lowercase
